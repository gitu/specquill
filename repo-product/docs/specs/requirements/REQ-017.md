# GitHub sign-in with gated access and admin bootstrap

Users SHALL be able to sign in with their GitHub account. A deployment SHALL
control who may enter and SHALL have a working administrator from its first
login onward.

> **REQ-017.1 · MUST** — GitHub sign-in SHALL be a first-class login
> provider beside generic OIDC and local accounts; the login page SHALL
> offer exactly the providers the deployment enables.

> **REQ-017.2 · MUST** — The resolved identity SHALL carry a stable subject
> (the GitHub account id), the display name, and a usable email — email is
> required for git authorship. Accounts hiding their profile email SHALL
> still resolve via their primary verified address.

> **REQ-017.3 · MUST** — An allow-list of GitHub handles SHALL gate sign-in;
> accounts outside it SHALL be refused with a clear message and no session.
> An empty allow-list (open access) is a deliberate operator choice.

> **REQ-017.4 · MUST** — Users whose email is configured as an administrator
> (`tenant.admin_emails`) SHALL hold the admin role in the config tenant
> from their first login — a fresh deployment is never left without a
> reachable management API.

> **REQ-017.5 · MUST** — Secrets (the OAuth client secret) SHALL reach the
> server only via environment variables named in the config, never as
> config-file values.
