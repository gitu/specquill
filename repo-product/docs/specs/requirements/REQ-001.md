# Protected default branch

The default branch of a project is the reviewed source of truth and MUST
never be edited directly.

> **REQ-001.1 · MUST** — Direct write operations against a protected branch
> SHALL be rejected with a machine-readable `protected_branch` error.

> **REQ-001.2 · MUST** — The first edit attempt on a protected branch SHALL
> transparently create or switch to the caller's personal workspace branch
> (`ws/<user>`), claimed server-side so two users can never share one.

> **REQ-001.3 · MUST** — Protected branches SHALL only advance through
> reviewed merges (pull requests with approvals pinned to the head commit).
