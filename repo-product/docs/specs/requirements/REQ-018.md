# Instant sync via push webhooks

A push to a registered repository's remote SHALL be reflected in the served
workspace immediately when a webhook is configured, with interval polling
remaining the guaranteed fallback.

> **REQ-018.1 · MUST** — The webhook endpoint SHALL authenticate every
> delivery by HMAC signature against a shared secret; unsigned or
> mis-signed requests SHALL be rejected without side effects.

> **REQ-018.2 · MUST** — A push event SHALL trigger a fetch of exactly the
> registered repositories whose remote is the pushed repository — never a
> blanket sync.

> **REQ-018.3 · MUST** — When the pushed branch is a writable project's
> default branch, the served state SHALL fast-forward to it; a diverged
> branch SHALL be left untouched (same rule as a manual pull).

> **REQ-018.4 · MUST** — Webhook processing failures SHALL never corrupt
> state: errors are logged and the polling interval remains the backstop.

> **REQ-018.5 · SHOULD** — Connected clients SHALL learn about
> webhook-triggered updates through the same event stream as any other
> repository change.
