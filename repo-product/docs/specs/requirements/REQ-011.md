# Project-scoped shareable URLs

Every location in the app SHALL be addressable by a URL that fully names it,
so links can be shared, bookmarked and replayed independently of any
per-browser state.

> **REQ-011.1 · MUST** — The URL SHALL encode the selected project. Opening a
> link SHALL activate the project it names, regardless of which project the
> visitor last used.

> **REQ-011.2 · MUST** — Switching projects SHALL be reflected in the URL, so
> browser history and back/forward navigation traverse project switches.

> **REQ-011.3 · MUST** — Locations SHALL be plain URL paths (no `#` fragment);
> the server SHALL resolve any deep link directly.

> **REQ-011.4 · SHOULD** — A URL naming an unknown or revoked project SHALL
> degrade gracefully to the default entry point, never to a broken view.
