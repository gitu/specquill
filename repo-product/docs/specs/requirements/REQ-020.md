# Per-repository user grants

Beyond roles derived from the git host, an administrator SHALL be able to
grant a user an explicit role on a single repository — a user outside the
GitHub repository (or outside GitHub entirely), an on-prem user without
git-host access, or a user whose git permission is lower than their intended
app role. The server pushes with its own credentials, so the application
layer is the effective gate.

> **REQ-020.1 · MUST** — A repo admin SHALL grant any role of the ladder
> ([REQ-021](REQ-021.md)) on one repository to a user identified by email
> or GitHub login. Grants compose with derived roles by maximum; they
> SHALL never be created, changed or revoked by a role sync.

> **REQ-020.2 · MUST** — Roles SHALL be enforced per repository on every
> request, following the ladder of [REQ-021](REQ-021.md): below `viewer`
> nothing is reachable; `viewer` reads and comments; mutations (writes,
> commits, PR create/approve/close, co-editing, copilot drafts) require
> `editor`; merging into a protected branch requires `maintainer`. A
> viewer SHALL NOT be able to write through any route.

> **REQ-020.3 · MUST** — In GitHub tenants the derived role SHALL be the
> user's permission on THAT repository (not the tenant maximum); the
> tenant-level role remains the maximum for tenant management. A permission
> lookup failure SHALL never revoke access mid-session.

> **REQ-020.4 · MUST** — A grant-only user SHALL see the tenant and exactly
> the granted repositories — no tenant management, no other repos.

> **REQ-020.5 · MUST** — Granting an identity with no account yet SHALL
> leave a pending invite (email or GitHub login, case-insensitive) that
> becomes a grant on the invitee's first login, whichever provider they use.

> **REQ-020.6 · SHOULD** — `tenant.default_role` SHALL control what
> authenticated users get in the config tenant: `editor` (default,
> self-host semantics), `viewer`, `maintainer`, or `none` — with `none`,
> access comes from grants alone (restricted on-prem deployments);
> configured admin emails still bootstrap as admin.
