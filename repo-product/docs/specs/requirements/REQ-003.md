# Projects in repository subfolders

A spec workspace and product code often share one repository. SpecQuill MUST
support opening a subfolder as a full project. (This very workspace lives in
`docs/specs/` of a monorepo — it is its own proof.)

> **REQ-003.1 · MUST** — A project SHALL be a git repository plus an optional
> `content_root` subfolder; all API paths are project-relative.

> **REQ-003.2 · MUST** — Content outside the project's `content_root` SHALL be
> invisible to reads and unreachable by writes, including via path traversal.

> **REQ-003.3 · MUST** — Commits made through a subfolder project SHALL touch
> only the project subtree.
