# Reviewed merges via pull requests

A workspace branch reaches the protected default branch only through a
reviewed, conflict-checked merge.

> **REQ-008.1 · MUST** — Merging a branch into a protected branch SHALL
> require an open pull request.

> **REQ-008.2 · MUST** — Approvals SHALL be pinned to the reviewed head
> commit; a new commit on the branch SHALL invalidate prior approvals.

> **REQ-008.3 · MUST** — Merges SHALL detect conflicts and refuse rather than
> produce a conflicted tree; the reference SHALL advance atomically so
> concurrent merges cannot clobber each other.

> **REQ-008.4 · SHOULD** — The commit SHALL record the author as author and
> committer, with the service identity and any collaborators as
> `Co-authored-by:` trailers.
