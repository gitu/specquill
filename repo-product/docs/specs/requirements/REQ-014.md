# Traceable document lifecycle

Documents move and evolve; their identity in git and their inbound references
SHALL survive it.

> **REQ-014.1 · MUST** — Moving or renaming a document SHALL happen as a git
> rename in the branch worktree, so committed history follows the file.

> **REQ-014.2 · MUST** — A move SHALL detect every document referencing the
> moved file (all link styles and typed frontmatter links) and offer to
> rewrite them to the new location.

> **REQ-014.3 · MUST** — A document's git history SHALL be viewable in place,
> following renames, with any listed version readable.
