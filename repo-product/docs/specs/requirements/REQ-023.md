# Encrypted tenant credentials for repo connection

A tenant admin SHALL connect repositories with a custom token (PAT) at
runtime. Tokens are tenant-owned secrets: encrypted at rest, write-only
after entry, and resolved through the same seam as GitHub App installation
tokens.

> **REQ-023.1 · MUST** — Credentials SHALL be named tenant-owned entities
> (label, optional username, token), attachable to any of the tenant's
> repositories and reference sources; one credential MAY back several
> repos. Deleting a credential still referenced SHALL be refused.

> **REQ-023.2 · MUST** — Tokens SHALL be sealed with authenticated
> encryption (AES-256-GCM) under a master key supplied by environment
> variable, with the ciphertext cryptographically bound to the owning
> tenant and a stored key id as the rotation seam. Without the master key
> the server boots normally and only the credential endpoints are
> unavailable.

> **REQ-023.3 · MUST** — A token SHALL never be echoed: list/read
> responses carry metadata only, rotation replaces without display, and
> logs/errors carry ids and labels — never token material.

> **REQ-023.4 · MUST** — Git credential resolution SHALL follow one
> precedence through a single seam: installation token, else the repo's
> attached tenant credential, else the configured environment fallback
> (`token_env`). Tokens SHALL reach git via child-process environment
> only — never argv or config files.

> **REQ-023.5 · MUST** — Connecting a repository SHALL accept an inline
> token that creates and attaches the credential in one step, and the
> initial clone SHALL already authenticate with it.
