# Non-git sources become mirror repositories

## Context

Reference knowledge also lives outside git — web pages, API specs, wikis.
These could be a separate storage and read path, or they could be normalized
onto the same git-backed model everything else uses.

## Decision

An importer mirrors each non-git source into a remote-less git repository
(see [importers.md](../specs/importers.md)). Downstream, a mirror is just a
read-only source: browsing, grounding and grant checks are identical to a git
source.

## Consequences

One read path, one grant model, and free history/diffing for imported
content. The cost is that imports must be idempotent full snapshots, and a
remote-less repo needs a bare-repo write path (no working copy) to commit.
