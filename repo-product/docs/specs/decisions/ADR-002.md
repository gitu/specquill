# Content roots map server-side

## Context

Subfolder projects need path mapping somewhere: in every client, or once in
the server.

## Decision

The server maps at the API boundary; clients always see project-relative
paths.

## Consequences

The client model, OKF tooling and collab bindings needed no changes; the
cost is a single choke point that every path-carrying handler must route
through — enforced by review and traversal tests.
