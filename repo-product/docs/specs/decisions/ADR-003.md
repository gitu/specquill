# The server is a dumb CRDT relay

## Context

Collaborative editing needs conflict resolution somewhere. The server could
merge document semantics, or it could stay content-agnostic and let a CRDT on
the clients converge.

## Decision

The server relays and durably logs opaque CRDT updates and never interprets
document content (see [co-editing.md](../specs/co-editing.md)). Convergence is
the CRDT's guarantee; the server only orders and replays.

## Consequences

The server has no document-model coupling and cannot corrupt a merge, but it
also cannot repair a client that mutates its document before replay is
applied — that discipline is the client's, enforced by the join protocol.
