# Postgres as the metadata store

## Context

Sessions, PR review state and collab update logs need durability across
instance replacement; content itself already lives in git.

## Decision

Use Postgres (Neon in production) for all non-content state. Git remains the
only store for documents.

## Consequences

Instances become disposable: clones and worktrees are caches, roomed drafts
replay from the update log, and only plain uncommitted drafts are at risk on
eviction.
