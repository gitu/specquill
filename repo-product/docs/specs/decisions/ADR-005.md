# Layered authorization with tenant-owned encrypted credentials

## Context

Access decisions were spread across seams that grew independently: three
role-rank maps, a hardcoded `default` tenant with its own enrollment and
bootstrap rules, tenant selection via a client-pinned header, merge rights
implied by write rights, and repo credentials that only an operator could
supply (environment variables at deploy time). Hosting many separate
organisations — and treating an on-prem installation as just another
organisation — needs one explicit model instead.

## Decision

Authorization is four stacked layers, each independently configured and
enforced top-down: **identity** (global users from GitHub/OIDC/local),
**tenancy & repo connection** (a tenant per GitHub App installation or per
on-prem installation, named in every URL; repos connect via installation
grants or tenant-owned credentials), **repo permissions** (one four-level
ladder `viewer < editor < maintainer < admin`, derived from the git host
and/or granted explicitly, with merge-into-protected at `maintainer`), and
**project configuration** (the in-repo config, which can only narrow).

Custom repo tokens become tenant-owned entities encrypted with
AES-256-GCM under an environment-supplied master key, bound to their
tenant, never echoed after entry, and resolved through the same single
`TokenFor` seam as installation tokens.

## Consequences

One ladder replaces three rank maps, and "who can merge" stops being "who
can write" — review can be broad while protected branches stay behind a
small group. URLs carry their full context, so links are unambiguous and
the client holds no tenant state to desynchronize. The config tenant loses
its special casing: the same enrollment, grants and credential machinery
serves on-prem and hosted. The costs: a four-value role vocabulary in
every surface (API, UI, grants), a master key the operator must manage
(losing it orphans stored tokens — they must be re-entered), and merge
rights that GitHub's `push` permission alone no longer confers.
